Akka, the actor-model

  • The Actor Model provides a higher level of abstraction for writing concurrent and distributed systems.
  • An actor is a container for State, Behavior, a Mailbox, Children and a Supervisor Strategy.

Actor reference: Actors are represented to the outside using actor references, which are objects that can be passed around freely and without restriction.

State: Actor objects will typically contain some variables which reflect possible states the actor may be in.

Behavior: Every time a message is processed, it is matched against the current behavior of the actor.
